# Quant Model Risk Lab

Quant Model Risk Lab is a public evidence repository focused on model validation, model monitoring, statistical risk modelling and financial-risk documentation.

The project demonstrates how financial models can be reviewed through independent validation logic, data-quality checks, sensitivity analysis, stress testing, model-output monitoring and clear documentation of assumptions and limitations.

The repository is not intended to represent proprietary banking models or confidential model-risk-management practices. It is an open, educational and professional portfolio project built with public data and reproducible Python workflows.

## Initial scope

- Model inventory and validation status tracking
- Data-quality checks for financial datasets
- Black-Scholes option-pricing validation
- Historical VaR and stress-testing workflows
- Model monitoring reports
- Documentation of assumptions, limitations and validation findings

## Why this repository exists

Model risk management is not only about building models. It is also about challenging assumptions, validating data, checking implementation logic, monitoring outputs, documenting limitations and producing evidence that a model remains fit for purpose.

This repository is designed as a professional portfolio project for quantitative finance, model validation and risk analytics roles.

## Current modules

| Module | Purpose |
|---|---|
| Data checks | Validate completeness, missing values and basic input quality |
| Black-Scholes | Demonstrate option-pricing validation logic and sensitivity checks |
| VaR backtesting | Estimate historical VaR and count model exceptions |
| Stress testing | Apply simple return shocks to portfolio values |
| Monitoring | Produce model monitoring summaries |
| Model inventory | Track simplified model lifecycle status |
