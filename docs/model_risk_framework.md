# Model Risk Framework

This document defines the open model-risk logic used in Quant Model Risk Lab.

## Core principle

A model should not be accepted only because it produces numbers. A model should be reviewed through its purpose, assumptions, data inputs, implementation logic, limitations, sensitivity and monitoring behaviour.

## Validation dimensions

1. Purpose and scope
2. Methodology and assumptions
3. Data quality
4. Implementation checks
5. Output reasonableness
6. Sensitivity analysis
7. Stress behaviour
8. Limitations
9. Monitoring requirements
10. Review status

## Independent challenge

The validation process asks whether the model is fit for its stated purpose, whether its assumptions are defensible, whether the implementation is reproducible and whether the outputs remain stable under changing market conditions.
