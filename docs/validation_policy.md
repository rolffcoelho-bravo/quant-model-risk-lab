# Validation Policy

This repository uses a simple validation policy for public and educational financial models.

## Minimum validation checks

- Input completeness
- Missing-value inspection
- Parameter reasonableness
- Output sanity checks
- Sensitivity to key assumptions
- Stress scenario response
- Known limitation documentation
- Monitoring status

## Validation result categories

- Pass
- Pass with limitations
- Under review
- Fail

## Important note

This repository does not validate proprietary banking models. It demonstrates public, reproducible model-risk logic using simplified financial models and open workflows.
